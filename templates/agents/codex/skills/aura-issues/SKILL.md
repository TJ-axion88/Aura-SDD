# aura-issues

## Goal

Read `.aura/specs/<feature>/tasks.md` and create GitHub Issues for each uncompleted task. Link issues to the spec and to each other via dependency relationships.

## Inputs

`aura-issues <feature-name> [--dry-run]`

## Prerequisites

- `gh` CLI installed and authenticated (`gh auth status`)
- `tasks.md` exists and is approved
- GitHub repository remote configured

## Execution Workflow

### Step 1 — Load tasks

Read `.aura/specs/<feature>/tasks.md`. Extract all unchecked tasks: `- [ ] **Task N.M**`.

### Step 2 — Dry-run preview (if --dry-run)

List all issues that would be created without actually creating them. Show title, labels, body preview.

### Step 3 — Create issues

For each task, create a GitHub Issue:

**Title format:** `[aura] <feature>: Task <id> — <title>`

**Labels:** `aura-sdd`, `spec:<feature>`, boundary label if distinct

**Body template:**
```markdown
## Task <id>: <title>

**Spec:** `.aura/specs/<feature>/spec.md`
**Plan:** `.aura/specs/<feature>/plan.md`
**Boundary:** `<boundary name>`
**Depends on:** <task ids or "none">
**Files:** `<file1>`, `<file2>`

### Requirements
<requirements mapping from tasks.md>

### Description
<task description from tasks.md>

---
_Generated by Aura-SDD `aura-issues`_
```

### Step 4 — Link dependencies

For tasks with `_Depends:_` on other tasks, add cross-references in issue bodies:
- "Blocked by #<issue-number>"

### Step 5 — Update tasks.md

For each created issue, add the issue URL to the task in tasks.md:

```markdown
- [ ] **Task 1.1** [P] — <title> ([#42](https://github.com/...))
```

### Step 6 — Summary

Report:
- N issues created
- Links to each issue
- Any tasks skipped (already complete or already has issue)

## Completion Criteria

- One GitHub Issue per uncomplete task
- Dependency relationships reflected in issue bodies
- tasks.md updated with issue links
