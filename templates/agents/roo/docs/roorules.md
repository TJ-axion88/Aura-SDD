# Aura-SDD v{{VERSION}} — Roo Code Integration

This project uses Aura-SDD for Spec-Driven Development with Boundary-First discipline and Constitutional Enforcement.

## Aura-SDD Commands

Commands are in `.roo/rules/aura/`. Reference with `/aura-<skill-name>`.

### Workflow
1. `/aura-constitution` — Establish project principles
2. `/aura-steering` — Build project memory
3. `/aura-discovery <idea>` — Route new work
4. `/aura-spec <feature>` — Requirements (EARS format)
5. `/aura-clarify <feature>` — Resolve ambiguity
6. `/aura-plan <feature>` — Architecture + Boundary Commitments
7. `/aura-tasks <feature>` — Task decomposition
8. `/aura-impl <feature>` — Autonomous implementation
9. `/aura-validate <feature>` — Integration validation

## Key disciplines

- **Boundary-First**: Identify and commit to boundaries before implementation
- **Constitutional Gates**: Check principles during planning
- **EARS format**: WHEN/IF/WHILE/WHERE/THE SYSTEM SHALL for all acceptance criteria
- **Phase gates**: Human approval at spec → plan → tasks → impl transitions
