# GEMINI.md — Aura-SDD v{{VERSION}}

This project uses Aura-SDD for Spec-Driven Development. Commands are in `.gemini/commands/aura/`.

## Aura-SDD Commands

```
/aura-discovery <idea>   → Route new work (start here)
/aura-constitution       → Project principles
/aura-steering           → Project memory
/aura-spec <feature>     → EARS-format requirements
/aura-clarify <feature>  → Resolve ambiguity
/aura-plan <feature>     → Architecture + Boundary Commitments
/aura-tasks <feature>    → Task decomposition
/aura-impl <feature>     → Autonomous implementation
/aura-validate <feature> → GO/NO-GO validation
```

## Disciplines

- Boundary-First: Contracts before implementation
- Constitutional Gates: `.aura/constitution.md` checked during planning
- EARS format: Structured acceptance criteria
- Phase gates: Human approval at each phase
