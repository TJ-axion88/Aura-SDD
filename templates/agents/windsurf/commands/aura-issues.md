# aura-issues

Read `.aura/specs/<feature>/tasks.md` and create GitHub Issues for each uncompleted task. Link issues to the spec and to each other via dependency relationships.

**Usage:** `/aura-issues <feature-name> [--dry-run]`

**Prerequisites:** `gh` CLI installed and authenticated (`gh auth status`), `tasks.md` exists and is approved, GitHub repository remote configured.

1. **Load tasks** — read `.aura/specs/<feature>/tasks.md`. Extract all unchecked tasks: `- [ ] **Task N.M**`.

2. **Dry-run preview (if --dry-run)** — list all issues that would be created without actually creating them. Show title, labels, body preview.

3. **Create issues** — for each task, create a GitHub Issue:

   **Title format:** `[aura] <feature>: Task <id> — <title>`

   **Labels:** `aura-sdd`, `spec:<feature>`, boundary label if distinct

   **Body template:**
   ```markdown
   ## Task <id>: <title>

   **Spec:** `.aura/specs/<feature>/spec.md`
   **Plan:** `.aura/specs/<feature>/plan.md`
   **Boundary:** `<boundary name>`
   **Depends on:** <task ids or "none">
   **Files:** `<file1>`, `<file2>`

   ### Requirements
   <requirements mapping from tasks.md>

   ### Description
   <task description from tasks.md>

   ---
   _Generated by Aura-SDD `/aura-issues`_
   ```

4. **Link dependencies** — for tasks with `_Depends:_` on other tasks, add cross-references in issue bodies: "Blocked by #<issue-number>".

5. **Update tasks.md** — for each created issue, add the issue URL to the task in tasks.md:
   ```markdown
   - [ ] **Task 1.1** [P] — <title> ([#42](https://github.com/...))
   ```

6. **Summary** — report: N issues created, links to each issue, any tasks skipped (already complete or already has issue).

**Completion criteria:** One GitHub Issue per uncomplete task, dependency relationships reflected in issue bodies, tasks.md updated with issue links.

Follow rules in `.aura/settings/rules/`.
